```markdown
# 测试编写指南

## 文件命名规范
- 单元测试: `ComponentName.test.ts`
- 集成测试: `ComponentName.integration.test.ts`
- 真实代码测试: `ComponentName-Real.test.ts`

## 测试结构标准
describe('组件名称测试', () => {
  describe('基础功能测试', () => {
    it('应该正确初始化', () => {
      // 测试代码
    });
  });
  
  describe('边界条件测试', () => {
    it('应该处理无效输入', () => {
      // 边界测试
    });
  });
  
  describe('错误处理测试', () => {
    it('应该抛出适当的错误', () => {
      // 错误测试
    });
  });
});

## 断言规范
- 使用语义化的expect断言
- 提供清晰的错误消息
- 验证关键属性和行为
```
